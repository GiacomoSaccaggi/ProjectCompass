API Routes Reference

All Flask routes defined in app.py

Home Overview Architecture API Routes Core Classes Templates Configuration Docker

All routes are served via Flask Blueprints (auth, catalog, data, agent, api). Authentication is enforced globally via @auth_bp.before_app_request: any unauthenticated request is redirected to the login page (except /health, /api/, and static files).

Dashboard & Navigation

RouteMethodDescriptionTemplate
/ GET Main dashboard. Reads all analyses, computes statistics (total count, owners, products, structured analyses), and renders a polar-area Chart.js chart of analyses by product. index.html
/overview GET Platform description page with inline HTML content describing the four core functionalities. Reuses index.html with intro=False. index.html
/todo GET Personal notes / to-do page. todopage.html
/documentation/ GET Embeds the GitHub repository page in an iframe. iframetemplate.html
/license/ GET Embeds the GitHub repository license page in an iframe. iframetemplate.html

Authentication

RouteMethodDescriptionTemplate
/login/ POST Validates uname and psw form fields against the ADMIN_PASSWORD_HASH environment variable using werkzeug. On success sets session['authenticated'] = True and redirects to dashboard. index.html
/reset/ GET Clears the Flask session. Returns {"success": true}. JSON

Analysis Catalog

RouteMethodDescriptionTemplate
/analysis GET Lists all analyses from Analyses/. Each entry is an accordion with metadata, links, and action buttons (open, edit, download, delete, run). analysis.html
/load_analysis/ GET Wrapper that embeds /load_analysis_internal/ in an iframe. Accepts optional ?name= query param for editing an existing analysis. iframetemplate.html
/load_analysis_internal/ GET Renders the analysis creation/edit form. If ?name= is provided, pre-fills the form with existing metadata. create_analysis.html
/produce_analysis/ POST Creates a new analysis (new_analysis=new) or updates an existing one. Calls webapp.upload_new_analysis() or webapp.modify_analysis(). Optionally commits to GitLab. process_complete.html / internal_error.html
/delete_analysis/ GET Deletes the analysis folder identified by ?name=. Uses shutil.rmtree. Optionally pushes deletion to GitLab. analysis.html
/open_analysis/ GET Packages the analysis folder as a ZIP archive and sends it as a file download. File download
/iframe/ GET Opens the dashboard link of an analysis. If the link contains <script it is injected as raw HTML; otherwise it is loaded in an iframe. iframetemplate.html
/physical_output/ GET Opens or downloads the physical output file of an analysis. HTML outputs are shown in an iframe; other formats are sent as file downloads. iframetemplate.html / File download
/open_version/ GET Shows the version management page for a single analysis (dashboard versions + output versions). versions_single_analysis.html
/add_version/ POST Adds a new dashboard link version or uploads a new output file version for an analysis. versions_single_analysis.html
/delete_version/ GET Removes a specific version (output file or dashboard link) from an analysis. versions_single_analysis.html

Structured Analysis Execution

RouteMethodDescriptionTemplate
/create_investigations/ GET Dynamically builds an HTML form from metadata_automatic_report.yaml input definitions. Supports text, datalist, radio, date, and other HTML5 input types. investigations.html
/run_investigation/ POST Executes structured_analysis_main.structured_analysis() from the analysis folder. Streams logs to a file, then returns the output (HTML iframe, file download, or download link). iframetemplate.html / File download
/loading/ GET Displays a loading spinner page used while an analysis is running. loader.html

Data Tools

RouteMethodDescriptionTemplate
/list_of_data/ GET Shows the data upload form. data_upload.html
/upload_data_file/ POST Accepts file uploads. Calls webapp.upload_text_files(request) which reads the file with pandas using the specified separator and header row, then saves it as a normalised CSV in Saved_data/. data_list.html
/all_data/ GET Lists all available datasets (CSV and SQLite files) in Saved_data/. data_list.html
/open_data GET Runs SELECT * FROM <name> via DuckDB and shows the result. Truncates to 2000 rows if the result is too large. query_results.html
/query_runner/ GET Opens the SQL editor with an empty result panel. query_results.html
/query_results/ POST Sanitises the submitted SQL (strips HTML tags from the rich-text editor), executes it via webapp.sql(), and returns the result table. query_results.html
/save_query/ POST Saves the query as a .sql file in Saved_queries/ with auto-extracted table names and a timestamp header. query_results.html
/tmp_table/ POST Executes a query and saves the result as a new CSV in Saved_data/ under the provided table name. query_results.html
/graph_analysis/ POST Executes a query and passes the CSV result directly to the RAWGraphs visualisation interface. RAWGraphs.html
/rawgraphs GET Opens the RAWGraphs visualisation page with an empty dataset. RAWGraphs.html

Utility Routes

Request Hooks

HookBehaviour
@app.before_request Checks session['authenticated']. Unauthenticated requests (except /login/) are redirected to login_page.html. When GitLab is enabled, performs a git reset --hard before every non-static request.
@app.after_request When GitLab is enabled, performs git add . && git commit && git push after every POST request to keep the remote in sync.
