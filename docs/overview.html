What is ProjectCompass?

ProjectCompass is a Python-based web application built on Flask that addresses the lifecycle management of data analyses and projects. After completing analytical work, three recurring problems emerge:

  • How to organize and categorize analyses in a way that is searchable across multiple dimensions.
  • How to provide accessible, standardized documentation for all stakeholders, technical and non-technical.
  • How to execute analyses consistently without environment compatibility issues.

ProjectCompass solves all three by combining a tag-based catalog, an interactive data exploration environment, a structured execution engine, and shareability tooling into a single unified platform.

Core Functionalities

📂 Catalog

Centralized repository of all analyses with metadata, tags, owners, products, and version history. Supports full-text and multi-dimensional search.

🔍 Data Exploration

Interactive SQL query runner powered by DuckDB. Run queries against uploaded CSV/SQLite files, save queries, and push results directly to RAWGraphs for visualization.

⚙️ Execution Engine

Launch structured analyses that produce physical outputs (HTML reports, CSV extractions, downloadable files) from a web form, with real-time logging.

🔗 Shareability

Download analysis packages as ZIP archives, embed dashboards via iframe, and manage versioned outputs and dashboard links per analysis.

Target Users

  • Data Scientists – catalog, document, and execute analytical workflows.
  • Research Teams – collaborate with standardized metadata and version control.
  • Project Managers – track analysis progress and manage deliverables.
  • Business Stakeholders – access results and documentation without technical setup.

Technology at a Glance

  • Backend: Python 3, Flask, Jinja2
  • Data layer: DuckDB (in-memory SQL on CSV/SQLite), pandas
  • Frontend: W3.CSS, Chart.js, NVD3/D3.js, jQuery, RAWGraphs
  • Security: werkzeug password hashing, AST-validated code sandbox, CORS, CSRF
  • Storage: File-based (Analyses/, Saved_data/, Saved_queries/)
  • Optional LLM: Ollama (llama3 + nomic-embed-text) via llama-index
