Quick Start

# Start both services
docker-compose up -d

# Follow logs
docker-compose logs -f

# Stop and remove containers
docker-compose down

Useful Docker Commands

# View application logs
docker logs projectcompass_app

# View Ollama logs
docker logs ollama

# Open a shell inside the app container
docker exec -it projectcompass_app bash

# Test Ollama API
curl http://localhost:11434/api/tags

# Stop containers without removing them
docker stop projectcompass_app ollama

# Remove containers
docker rm projectcompass_app ollama

# Remove the Ollama model volume (WARNING: deletes all downloaded models)
docker volume rm ollama_data

Health Check

The application exposes a health check endpoint at GET /check which returns {"success": true} with HTTP 200. The .dockerhealthcheck file configures Docker to poll this endpoint.

curl http://localhost:8080/check

Production Considerations

Set SECRET_KEY and ADMIN_PASSWORD_HASH environment variables before deploying. Generate a password hash with: python -c "from werkzeug.security import generate_password_hash; print(generate_password_hash('your-password'))"
  • RAM: Ollama with llama3 requires at least 8 GB of available RAM.
  • Disk: LLM model weights require several GB of disk space in the ollama_data volume.
  • Ports: Ensure ports 8080 and 11434 are not blocked by a firewall.
  • Backups: Regularly back up the ./Analyses directory and the ollama_data volume.
  • CORS: Update ALLOWED_ORIGINS in config.py or via environment variable to restrict cross-origin access.
  • Gunicorn: The production image uses Gunicorn as the WSGI server. Tune workers and timeout in docker-compose.prod.yml as needed.
