constants.yaml

Path: templates/environment_variables/constants.yaml
Loaded by HTMLUtils.__init__() into self.constants. Read on every application start.

KeyDefaultDescription
port5000Port the Flask development server listens on. Overridden by the PORT environment variable.
debugFalseFlask debug mode flag.
logo_svg(SVG string)Inline SVG used as the application logo in download/result pages.

tool_choices.yaml

Path: templates/environment_variables/tool_choices.yaml
Loaded into self.tool_choices. Drives all dropdown and datalist options in the UI.

KeyDescription
productList of product categories shown in the analysis filter and upload form.
countriesList of country options for the analysis filter and upload form.
input typeAllowed variable types for analysis inputs (string, double, date, integer).
output typeAllowed output type descriptions for an analysis.

To add a new product or country, append a value to the relevant list and restart the application.

links.yaml

Path: templates/environment_variables/links.yaml
Loaded into self.environments_info['links']. Rendered as a list of anchor tags on the dashboard.

ProjectCompass: https://github.com/GiacomoSaccaggi/ProjectCompass
Giacomo Saccaggi: https://giacomosaccaggi.github.io/

Add any key-value pair to expose a new link on the dashboard sidebar.

intro.txt and title.txt

Path: templates/environment_variables/
Plain text files read into self.environments_info['intro'] and self.environments_info['title']. Edit these to customise the dashboard welcome text and the browser/page title without touching any Python code.

Analysis metadata.yaml Structure

Each analysis folder contains a metadata.yaml file written by upload_new_analysis():

title: My Analysis
start_date: 2024-01-15
description:
    Detailed description of the analysis purpose.
owner: owner@example.com
collaborators: collaborator@example.com
product: Research project
countries: Italy
inputs:
    start_month:
        default: "2024-01"
        type: date
links:
    Report: https://example.com/report
gitlab: https://gitlab.com/repo
gdrive: https://drive.google.com/folder
dashboard link: "https://example.com/dashboard"
output_type: Static report
output description:
    Description of what the output contains.

metadata_automatic_report.yaml (Structured Analyses)

Present in Analyses/<name>/analysis/ for analyses that support the execution engine. Defines the input form fields:

input:
  start_date:
    HTML_input_type: date
  environment:
    HTML_input_type: radio
    list:
      - production
      - research
  region:
    HTML_input_type: datalist
    list:
      - EU
      - US

Supported HTML_input_type values: text, date, month, datetime-local, week, time, number, email, password, tel, url, color, radio, datalist.
