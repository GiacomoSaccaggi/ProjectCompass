Header Injection System

Rather than Jinja2 template inheritance, ProjectCompass uses a custom partial-injection approach. header.html is parsed once at startup by HTMLUtils.__init__() using HTML comment markers: 

<!-- start head -->        ...CSS/JS includes...   <!-- end head -->
<!-- start top_container -->  ...top bar HTML...   <!-- end top_container -->
<!-- start sidebar_menu -->   ...sidebar HTML...   <!-- end sidebar_menu -->

Each section is stored in self.html_parts. Before every render, substitute_html(port=..., project_folder=...) replaces {{ port }} and {{ project_folder }} placeholders and returns the three rendered strings, which are passed as head, top_container, and sidebar_menu to render_template().

Every template injects them with the |safe filter:

{{ head|safe }}
{{ top_container|safe }}
{{ sidebar_menu|safe }}

Common Variables

These variables are passed to almost every template:

VariableTypeDescription
headstr (HTML)Rendered <head> section with CSS/JS includes
top_containerstr (HTML)Top navigation bar
sidebar_menustr (HTML)Left sidebar with navigation links
titlestrPage title from title.txt
portintApplication port (from constants.yaml)
project_folderstrURL-safe project folder prefix (e.g. ProjectCompass/)
introstr or FalseIntro text from intro.txt; False hides the stats block
linksdictUseful links from links.yaml

Template Reference

index.html – Dashboard

The main landing page. When intro is truthy it renders four KPI cards (analyses count, owners, structured analyses, products) and a polar-area Chart.js chart of analyses by product.

VariableDescription
intro_titleRaw HTML injected above the stats (hero image or overview text)
number_of_analysisTotal analyses count
number_of_ownersDistinct owner count
number_of_structured_analysisAnalyses with execution engine enabled
number_of_productsDistinct product count
product_labelsJS-safe comma-separated quoted product names
product_resComma-separated counts per product
product_color1/2Randomly generated RGBA colour strings for the chart

analysis.html – Analysis Catalog

Renders an accordion list of all analyses with search/filter controls (text, product, country, date range). Each accordion panel shows full metadata and action links.

VariableDescription
analysisDict of analysis dicts keyed by folder name (from read_multiple_analysis())
tool_choicesDict from tool_choices.yaml used to populate datalist dropdowns

The client-side search_function() filters accordion items in real time across name, description, product, country, and date fields without any server round-trip.

create_analysis.html – Upload / Edit Form

Multi-field form for creating or editing an analysis. Pre-filled when new_analysis=False.

VariableDescription
new_analysisBoolean – controls form mode (create vs edit)
analysisExisting analysis dict for pre-filling (empty dict for new)
tool_choicesDropdown options for product, countries, output type

query_results.html – SQL Runner

Rich-text SQL editor (with syntax highlighting) and a results table. Action buttons send the query to /query_results/, /save_query/, /tmp_table/, or /graph_analysis/.

VariableDescription
queryPre-filled SQL string
dfpandas DataFrame with query results
columnsColumn names list
show_res'block' or 'none' – controls result panel visibility
messageStatus/error message (e.g. truncation warning)

data_upload.html – File Upload

Form for uploading CSV/TSV/text files. Accepts separator and header-row configuration.

data_list.html – Data List

Lists all available datasets with links to open them in the SQL runner.

VariableDescription
all_dataList of dataset names from list_data()

investigations.html – Execution Form

Dynamically generated form for running a structured analysis. Input fields are built server-side from metadata_automatic_report.yaml.

VariableDescription
input_formList of HTML input strings injected into the form

iframetemplate.html – Iframe Wrapper

Generic wrapper that either embeds a URL in an <iframe> or injects raw HTML (for embedded dashboards containing <script> tags).

VariableDescription
urlURL to load in the iframe (empty if using obj)
objRaw HTML to inject directly (empty if using url)
titlePage title / direct link anchor

versions_single_analysis.html – Version Manager

Shows all dashboard link versions and physical output versions for one analysis, with add/delete controls.

VariableDescription
valuesFull analysis dict from read_single_analysis()
nomeAnalysis folder name

RAWGraphs.html – Visualisation

Embeds the RAWGraphs charting library. When input_data is provided (CSV string from a query), it is pre-loaded into the tool.

VariableDescription
input_dataCSV string with newlines escaped as \n
