HTMLUtils utils/html_utils.py

Base class in the inheritance chain. Responsible for loading all environment configuration and parsing the shared header.html partial into named sections.

Constructor

HTMLUtils(dir_path: str, **kwargs)

On initialisation it:

  • Reads templates/environment_variables/constants.yaml → self.constants
  • Reads templates/environment_variables/tool_choices.yaml → self.tool_choices
  • Reads intro.txt, links.yaml, title.txt → self.environments_info
  • Parses header.html using HTML comment markers (<!-- start X --> / <!-- end X -->) into self.html_parts dict
  • Overrides self.constants['port'] from the PORT environment variable if set

Key method

substitute_html(**kwargs) → dict

Iterates over self.html_parts and replaces {{ key }} placeholders with the provided keyword arguments. Returns a dict of rendered HTML strings (e.g. head, top_container, sidebar_menu) ready to be passed to render_template().

DataUtils utils/data_utils.py

Handles all data file operations and SQL execution. Inherits from HTMLUtils.

Constructor

DataUtils(dir_path: str, **kwargs)

Sets self.save_queries_path and self.save_data_path, creating both directories if they do not exist.

Methods

sql(sql_query: str) → pd.DataFrame

Opens an in-memory DuckDB connection. Scans Saved_data/ for CSV files and maps each filename (without extension) to a read_csv_auto('/path/file.csv') AS alias expression. SQLite/DB files are attached as separate schemas via ATTACH … (TYPE sqlite). The user query is rewritten with these substitutions and executed. The connection is always closed in a finally block. Returns an empty DataFrame on error.

upload_text_files(request)

Iterates over request.files.getlist('upload_files'). Each file is saved temporarily, read with read_data() using the form-supplied separator and header row, then written as a normalised CSV. The temporary file is removed afterwards.

list_data() → list[str]

Returns the names (without extension) of all .csv, .sqlite, and .db files in Saved_data/.

read_data(file_path, separator, header) → pd.DataFrame [static]

Thin wrapper around pd.read_csv(). Normalises the separator (\\t → tab, empty → comma) and the header row (-1 → None). Uses the Python engine for compatibility with complex delimiters.

AnalysisUtils utils/analysis_utils.py

Manages the analysis catalog stored under Analyses/. Inherits from DataUtils.

Constructor

AnalysisUtils(dir_path: str, **kwargs)

Sets self.save_folder_path = dir_path + 'Analyses' and creates the directory.

Methods

read_single_analysis(main_folder, folder) → dict

Reads metadata.yaml and output_versions.yaml for one analysis folder. Enriches the dict with computed fields: last_modified, gitlab_link_viz, gdrive_link_viz, inline_inputs, inline_links, embedded, physical_output, versions_output, versions_dashboard. Detects structured analyses by checking for structured_analysis_main.py and metadata_automatic_report.yaml in the analysis/ subfolder. Also regenerates the per-analysis readme.md.

read_multiple_analysis() → dict

Lists all non-hidden subdirectories of Analyses/ and calls read_single_analysis() for each. Returns a dict keyed by folder name.

upload_new_analysis(request) → str

Creates the full folder structure for a new analysis (analysis/, physical_output/main/, sql_extractions/), writes metadata.yaml and output_versions.yaml, saves uploaded files, auto-generates API documentation with pdoc, extracts import statements into requirements.txt, and extracts SQL queries from docstrings into sql_extractions/. Returns an error string (empty on success).

modify_analysis(request) → str

Overwrites metadata.yaml and updates the main version in output_versions.yaml for an existing analysis. Replaces the main physical output file if a new one is uploaded.

produce_readme()

Regenerates the top-level Analyses/readme.md with a summary table of all analyses (product, title, owner).

make_archive(source, destination) [static]

Creates a ZIP archive of a source directory and moves it to the destination path.

ProjectCompass basefun.py

Top-level class. Inherits from DataUtils, AnalysisUtils, and HTMLUtils (via the MRO chain). Adds project-folder path resolution and the recipe job-request builder.

Constructor

ProjectCompass(dir_path: str)

Resolves self.project_folder (the URL-safe last path segment of dir_path) and populates self.environments with the template and analysis folder paths.

create_job_request(RECIPE_ID, RECIPE_VERSION, **kwargs) → dict [static]

Builds a v2 job-request payload for the Intelligence Studio Recipe API. Accepts arbitrary keyword arguments under input_recipe.

Crypto Helpers basefun.py

encrypt_and_compress(text, key='key') → str

XOR-encrypts the text with the key, compresses with zlib, and encodes with Base64. Used to protect sensitive configuration values.

decrypt_and_decompress(encrypted_string, key='key') → str

Reverses the above: Base64 decode → zlib decompress → XOR decrypt.
