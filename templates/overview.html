ProjectCompass

Analysis catalog, execution engine, and data exploration platform

What is ProjectCompass?

ProjectCompass is a Python-based tool for teams that produce data analyses. It solves three problems: organization (a tag-based catalog with metadata and versioning), execution (a unified environment to run structured analyses), and exploration (SQL queries on local datasets via DuckDB, with visualization and AI chat).

Features

Analysis Catalog

Tag-based repository with search, filter by product/owner/country, pagination, and version tracking. Each analysis has metadata, documentation, queries, and physical outputs.

Data Explorer

SQL queries on CSV/SQLite via DuckDB. Upload datasets, run queries with syntax highlighting, save queries, and paginate results.

RAWGraphs Visualization

Send query results directly to RAWGraphs for interactive data visualization. Supports alluvial diagrams, bar charts, treemaps, and more.

Execution Engine

Run structured analyses with dynamic forms generated from YAML metadata. Produces physical outputs (HTML reports, data files).

AI Chat

Ask questions about your data in natural language. Uses Ollama + LLaMA with AST-validated safe code execution (sandboxed pandas).

REST API

JSON endpoints for analyses, data, and queries. Session-based auth, CORS-restricted. Useful for integrations and automation.

Key Design Decisions

AspectChoiceRationale
DatabaseDuckDB (in-memory)SQL on CSV without loading into memory; zero infrastructure
StorageFile-basedAnalyses stored as folders with YAML metadata; no external DB needed
AuthSession + werkzeug hashSimple, secure; no OAuth complexity for internal tools
AIOllama (local LLM)Privacy-first; no data leaves the server; lazy-loaded
Code safetyAST validationLLM-generated pandas code is validated against a whitelist before execution
FrontendW3.CSS + Jinja2Lightweight, no build step, responsive out of the box

Quick Start

Docker (fastest)

docker compose -f docker-compose.public.yml up -d

Open localhost:8080. Pull LLM models optionally with docker exec ollama ollama pull llama3.

Local Development

uv sync && cp .env.example .env
uv run python app.py

Open localhost:5000. Tests: uv run pytest -v

Data Flow

1 Upload CSV/TSV files → stored in Saved_data/
2 Query files via DuckDB SQL → results in table
3 Visualize results in RAWGraphs or Chart.js
4 Ask AI questions about datasets in natural language
5 Catalog analyses with metadata, tags, outputs, and versions
